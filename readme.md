# Red Black Tree Unit Tests

Проект с unit-тестами для реализации красно-черного дерева на C#/.NET.

Основная цель проекта — проверить корректность работы структуры данных, включая вставку элементов, обработку дубликатов, выборку данных, балансировку дерева и поведение в исключительных ситуациях.

## Стек технологий

- C#
- .NET 7
- NUnit
- Moq
- Coverlet
- Visual Studio Test Explorer

## Что тестируется

В проекте реализованы автотесты для проверки ключевой логики красно-черного дерева:

- создание дерева;
- вставка новых элементов;
- обработка повторяющихся значений;
- поиск и выборка данных по условию;
- корректность структуры после добавления элементов;
- балансировка дерева после вставки;
- работа левых и правых поворотов;
- проверка цвета корневого и дочерних узлов;
- обработка исключительных и граничных случаев.

## Основные навыки, которые демонстрирует проект

- написание unit-тестов;
- применение базовых техник тест-дизайна;
- проверка позитивных и негативных сценариев;
- тестирование граничных случаев;
- использование mock-объектов;
- изоляция тестируемой логики;
- работа с интерфейсами и классами;
- применение принципов ООП;
- анализ покрытия кода тестами.

## Запуск тестов

Склонировать репозиторий:

```bash
git clone https://github.com/username/red-black-tree-unit-tests.git
cd red-black-tree-unit-tests
```

Восстановить зависимости:

```bash
dotnet restore
```

Запустить тесты:

```bash
dotnet test
```

Запустить тесты с отчетом о покрытии:

```bash
dotnet test /p:CollectCoverage=true
```

## Примеры проверяемых сценариев

### Вставка элемента

Проверяется, что после добавления нового значения элемент появляется в дереве, а структура остается корректной.

### Обработка дубликатов

Проверяется поведение дерева при попытке добавить значение, которое уже существует.

### Балансировка дерева

Проверяется, что после вставки нескольких элементов дерево сохраняет свойства красно-черного дерева.

### Повороты дерева

Тестируются ситуации, в которых необходимы левый или правый поворот для восстановления баланса.

### Исключительные ситуации

Проверяется корректная обработка некорректных или граничных входных данных.

## Почему проект полезен с точки зрения тестирования

Проект показывает базовые навыки автоматизированного тестирования программной логики. В нем тестируется не только стандартное поведение структуры данных, но и сложные внутренние сценарии: балансировка, повороты, цвета узлов и обработка ошибок.

Такой проект демонстрирует понимание:

- как писать автотесты;
- как изолировать тестируемую логику;
- как проверять алгоритмическую корректность;
- как использовать mock-объекты;
- как работать с unit-тестами в .NET-проекте.
